## 2. 자료구조 비교표

### R01 설계 vs 각 구현

| 항목 | R01 설계 (ACCNode) | acc_core.py | acc_core_new.py | acc_core_acc2.py |
|------|-------------------|-------------|-----------------|------------------|
| **자료구조 타입** | 트리 클래스 (`ACCNode`) | 입력: 트리 (`DendroNode`), 작업: flat dict list | 입력: matrix dict, 작업: cluster dict + `structure` dict | 입력: matrix dict, 작업: `levels` list + `merge_points` dict |
| **트리 유지 여부** | 전체 과정에서 트리 유지 | 입력 트리에서 추출 후 flat list로 변환 → **트리 소실** | `structure` 필드에 중첩 dict로 트리 보존 → **트리 유사** | 트리 없음, 레벨 기반 → **트리 미보유** |
| **노드 정보** | `local_sim`, `global_sim`, `diameter`, `angle` 모두 노드에 저장 | flat dict에 저장 (노드 아닌 리스트 원소) | `structure` dict에 `angle`, `radius` 저장, `sim`은 cluster dict에 | `levels` dict에 `local_sim`, `global_sim`, `diameter`, `radius` 저장 |
| **Leaf 표현** | `ACCNode(members={"J"})` | `DendroNode(["J"], sim=1.0)` — 입력 트리에만 존재 | 문자열 `"J"` — structure 내 | 문자열 `"J"` — positions dict의 키 |
| **Internal Node** | `ACCNode(left, right, ...)` | flat dict `{"members": {...}, "sim_local": ...}` | `{"children": [left, right], "angle": ..., "radius": ...}` | level dict `{"cluster1": ..., "cluster2": ..., "local_sim": ...}` |
| **부모-자식 관계** | `node.left`, `node.right` | 없음 (flat list) | `structure["children"]` | `level["cluster1"]`, `level["cluster2"]` |

### 자료구조 상세 비교

#### acc_core.py — Cluster Dict

```python
# DendroNode (입력 전용)
class DendroNode:
    members: set          # {"J", "T"}
    sim: float            # 0.9
    left: DendroNode      # 자식
    right: DendroNode     # 자식

# Cluster Dict (작업용 — flat list의 원소)
cluster = {
    "members": {"J", "T"},       # 멤버 집합
    "sim_local": 0.9,            # Local 유사도
    "sim_global": 0.88,          # Global 유사도
    "diameter": 1.136,           # unit / sim_global
    "theta": 18.0,               # 180° × (1 - sim_local)
    "center": (0.0, 0.0),        # 원 중심
    "points": {"J": (x,y), ...}, # 멤버별 좌표
    "midline_angle": 0.0,        # 기준 각도
}
```

#### acc_core_new.py — Cluster Dict + Structure

```python
# Cluster Dict (작업용)
cluster = {
    "members": {"J", "T"},
    "center": (0.0, 0.0),
    "radius": 0.568,              # diameter / 2
    "diameter": 1.136,
    "angle": 18.0,                # theta
    "local_sim": 0.9,
    "global_sim": 0.88,
    "points": {"J": (x,y), ...},
    "midline_angle": 0.0,
    "structure": {                 # ← 트리 구조!
        "children": ["J", "T"],
        "angle": 18.0,
        "radius": 0.568
    },
}
```

#### acc_core_acc2.py — Levels + Merge Points

```python
# Level Dict (merge 순서)
level = {
    "level": 1,
    "cluster1": "O",                # 첫 번째 자식 ID
    "cluster2": "Q",                # 두 번째 자식 ID
    "members": ["O", "Q"],          # sorted list
    "structure": ["O", "Q"],        # 자식 ID list
    "local_sim": 0.85,
    "global_sim": 0.83,
    "diameter": 1.17,               # 1 + (1 - global_sim) [ACC2 공식]
    "radius": 0.585,
}

# Merge Point Dict
merge_point = {
    "radius": 0.585,
    "angle": 45.0,                  # 두 자식 중간 각도
    "children": ["O", "Q"],
    "level": 1,
}

# Position Dict (area별)
position = {
    "x": 0.354,
    "y": 0.354,
    "radius": 0.5,                  # 모든 area가 r=0.5
    "angle": 45.0,
}
```

## 3. 프로세스 비교표

### R01의 3단계 vs 각 구현의 파이프라인

| 단계 | R01 설계 | acc_core.py | acc_core_new.py | acc_core_acc2.py |
|------|---------|-------------|-----------------|------------------|
| **Phase 1: 트리 구축** | `build_tree_from_dendro()` — DendroNode → ACCNode 트리 변환 | `extract_clusters_from_dendro()` — DendroNode DFS → **flat list** 추출. 트리 구조 버림 | 해당 없음 (dendrogram 미사용). Matrix에서 직접 highest pair 찾아 incremental 구축 | `analyze_dendrogram_levels()` — Matrix에서 UPGMA 수행 → **levels list** 생성 |
| **Phase 2: 속성 계산** | `decorate_tree()` — global_sim/diameter/angle을 트리 노드에 기록 | `decorate_clusters()` — flat list 순회하며 각 dict에 sim_global/diameter/theta 기록 | `place_first_two_areas()` + `add_area_to_cluster()` 내에서 매번 sim 계산 + structure dict 생성 | `analyze_dendrogram_levels()` 내에서 level마다 global_sim, diameter, radius 계산 |
| **Phase 3: 좌표 계산** | `compute_coordinates()` — 트리 전위 순회, 루트→리프 | `place_first_cluster()` → `add_area_to_cluster()` / `merge_two_clusters()` — **구축과 동시에** 좌표 계산 | `position_structure_recursively()` — structure dict 재귀 순회 → 좌표 계산. **Phase 2와 혼재** | `calculate_final_positions()` — ACC1 결과에서 각도 추출 → `calculate_merge_points()` |
| **분리도** | 완전 분리 (3단계 독립) | Phase 2까지 분리, Phase 3는 병합과 혼재 | Phase 2+3 혼재 (매 병합마다 좌표 재계산) | 부분 분리 (levels 계산 → 위치 계산 → 연결선) |

## 4. 파일별 상세 분석

### 4.1 acc_core.py

**입력**: `DendroNode` 트리 (Local + Global) + Global similarity matrix

**핵심 특징**: 유일하게 **dendrogram 트리를 입력으로 받는** 구현

**파이프라인**:

```
DendroNode (Local)
    │
    ▼
extract_clusters_from_dendro()  ← DFS로 모든 노드를 flat list로 추출
    │                              *** 여기서 트리 구조 소실 ***
    ▼
clusters = [{members, sim_local, ...}, ...]  ← flat list
    │
    ▼
decorate_clusters()  ← global_sim, diameter, theta 계산
    │
    ▼
sort by sim_local DESC
    │
    ▼
place_first_cluster()  ← 첫 번째 클러스터 좌표 배치
    │
    ▼ (반복)
add_area_to_cluster() 또는 merge_two_clusters()
    │     ← 병합 + 좌표 계산 동시 수행
    ▼
최종 결과 (단일 cluster dict)
```

**R01과의 차이**:

- DendroNode는 트리이지만, `extract_clusters_from_dendro()`에서 flat list로 변환됨
- 이후 과정에서 "어떤 클러스터가 어떤 자식에서 파생되었는지"를 알 수 없음
- 병합 순서는 sim_local 정렬에 의해 결정되지만, 원래 dendrogram 구조와 반드시 일치하지는 않음
- `build_acc_merged`에서 병합 판단: `len(c["members"]) == len(base["members"]) + 1 and base["members"].issubset(c["members"])` — **멤버 수 비교**로 add vs merge 결정

**장점**:
- 코드가 간결 (531줄)
- 입력이 DendroNode 트리 → 외부에서 구조 제어 가능

**단점**:
- 트리 구조 소실로 계층 관계 재현 불가
- 좌표 재계산 시 전체 파이프라인 재실행 필요
- 병합과 좌표 계산 혼재

### 4.2 acc_core_new.py

**입력**: Similarity matrix (Local + Global) — dict of dict

**핵심 특징**: **`structure` dict가 사실상 트리 역할**. 3개 구현 중 R01 설계에 가장 가까움.

**파이프라인**:

```
Local/Global similarity matrix
    │
    ▼
find_highest_similarity_pair()  ← 가장 높은 유사도 쌍 찾기
    │
    ▼
place_first_two_areas()  ← 첫 쌍 배치 + structure 생성
    │                       structure = {"children": ["J","T"], "angle": 18, "radius": 0.57}
    ▼ (반복)
find_highest_similarity_with_clusters()  ← 다음 병합 대상 탐색
    │
    ├── new_pair → place_independent_pair()
    ├── add_to_cluster → add_area_to_cluster()
    │     structure = {"children": [old_structure, "Y"], "angle": 36, "radius": 0.64}
    └── merge_clusters → merge_two_clusters()
          structure = {"children": [c1_structure, c2_structure], "angle": 72, "radius": 0.91}
    │
    ▼
position_structure_recursively()  ← structure dict를 재귀 순회하여 좌표 계산
    │
    ▼
최종 결과 (steps list, 각 step에 cluster snapshots)
```

**Structure dict의 트리 구조**:

```python
# 최종 structure (예시)
{
    "children": [
        {  # JTY 하위 트리
            "children": [
                {"children": ["J", "T"], "angle": 18.0, "radius": 0.568},
                "Y"
            ],
            "angle": 36.0,
            "radius": 0.641
        },
        {  # NOQ 하위 트리
            "children": [
                "N",
                {"children": ["O", "Q"], "angle": 27.0, "radius": 0.602}
            ],
            "angle": 45.0,
            "radius": 0.714
        }
    ],
    "angle": 72.0,
    "radius": 0.909
}
```

**R01과의 유사점**:
- `structure` dict가 중첩 구조로 트리를 표현 → ACCNode 트리와 개념적으로 동일
- `position_structure_recursively()`가 R01의 Phase 3 (루트→리프 좌표 계산)과 대응
- 각 내부 노드에 `angle`, `radius`를 저장 → R01의 `angle`, `diameter/2`에 대응

**R01과의 차이점**:
- `structure`에 `local_sim`, `global_sim`을 **저장하지 않음** → cluster dict에만 최상위 값 존재
- Phase 2와 Phase 3가 분리되지 않음: 매 병합 시 `add_area_to_cluster()` 내에서 sim 계산 + structure 생성 + 좌표 계산을 한꺼번에 수행
- 트리가 dict로 표현되어 메서드/프로퍼티 없음 (단순 데이터)
- dendrogram 미사용: matrix에서 직접 UPGMA-like 과정 수행

**장점**:
- 트리 구조 보존 (`structure`)
- 좌표 계산이 `position_structure_recursively()`로 분리되어 재계산 가능
- 다중 독립 클러스터 처리 지원

**단점**:
- sim 정보가 structure에 없어 나중에 복원 불가
- 코드가 길고 복잡 (~1,160줄)
- Phase 분리가 불완전

### 4.3 acc_core_acc2.py

**입력**: Similarity matrix (Local + Global)

**핵심 특징**: **dendrogram을 동심원에 투영**하는 별도 알고리즘. ACC1과 다른 목적.

**파이프라인**:

```
Local/Global similarity matrix
    │
    ├─────────────────────────┐
    ▼                         ▼
build_acc_iterative()     analyze_dendrogram_levels()
(acc_core_new.py)         (자체 UPGMA 구현)
    │                         │
    ▼                         ▼
ACC1 final positions      levels list
(area별 각도 결정)        (merge 순서 + 반지름)
    │                         │
    ▼                         ▼
positions dict            merge_points dict
(area → angle, r=0.5)    (cluster_id → radius, angle)
    │                         │
    └────────┬────────────────┘
             ▼
    generate_connection_lines()
             │
             ▼
    {levels, positions, merge_points, lines, circles}
```

**R01과의 차이**:
- **목적이 다름**: R01은 ACC1의 트리 기반 재설계, ACC2는 dendrogram→동심원 투영
- 트리 자체를 구축하지 않고 `levels` (병합 이력 리스트)로 관리
- `structure`는 acc_core_new.py의 결과를 재활용하지만, 자체적으로 `parse_structure()`로 역분석
- diameter 공식이 다름: `1 + (1 - sim_global)` vs `unit / sim_global`

**자료구조 특성**:
- `levels`: 병합 순서의 **시간축** 표현 (1, 2, 3, ...). 트리의 **depth**와는 다른 개념
- `merge_points`: 병합 지점의 (반지름, 각도). 트리 노드가 아닌 **렌더링 좌표**
- `positions`: 모든 area가 r=0.5에 배치 — ACC1의 위치와 근본적으로 다름

**장점**:
- ACC1과 ACC2의 관심사가 명확히 분리됨
- 연결선(radial + arc) 생성이 체계적

**단점**:
- acc_core_new.py에 의존 (build_acc_iterative 호출)
- structure 역분석(parse_structure)이 취약 — sim 값을 역산해야 함
- 자체 UPGMA(analyze_dendrogram_levels)와 acc_core_new.py의 구축이 독립적으로 수행되어 불일치 가능

## 6. R01 설계에 가장 가까운 구현

**acc_core_new.py**가 R01 설계에 가장 가까움. 이유:

1. **`structure` dict가 사실상 트리** — R01의 ACCNode 트리와 개념적으로 대응
2. **`position_structure_recursively()`** — R01의 Phase 3 (전위 순회 좌표 계산)과 거의 동일한 패턴
3. **각 내부 노드에 angle, radius 저장** — R01의 노드별 기하 파라미터 저장과 유사

### R01 실현을 위한 격차 (acc_core_new.py 기준)

| 격차 | 설명 | 해소 방안 |
|------|------|-----------|
| **sim 미저장** | `structure` dict에 `local_sim`, `global_sim`이 없음 | structure에 sim 필드 추가 |
| **Phase 분리 부족** | 매 병합 시 sim 계산 + structure 생성 + 좌표 계산 동시 수행 | 트리 구축(Phase 1+2)과 좌표 계산(Phase 3) 분리 |
| **dict vs class** | structure가 plain dict → 타입 안전성/메서드 없음 | ACCNode 클래스 도입 또는 dataclass 활용 |
| **dendrogram 미활용** | matrix에서 직접 구축 → 외부 dendrogram 주입 불가 | Phase 1에서 DendroNode → ACCNode 변환 경로 추가 |

## 7. 결론 및 권장사항

### 현황 요약

- **acc_core.py**: 원본. 간결하지만 트리 소실. DendroNode 입력 가능이라는 독자적 장점 보유
- **acc_core_new.py**: structure dict로 트리를 유지하며 R01 설계에 가장 근접. 리팩토링 기반으로 적합
- **acc_core_acc2.py**: ACC2 전용. R01과는 목적이 다르므로 별도 유지 타당

### 권장 접근

1. **acc_core_new.py를 기반으로** R01의 ACCNode 트리 구조를 구현
   - `structure` dict를 ACCNode 클래스로 격상
   - 각 노드에 `local_sim`, `global_sim` 추가 저장
   - Phase 2 (트리 완성)와 Phase 3 (좌표 계산)을 명확히 분리

2. **acc_core.py의 DendroNode 입력 경로를 보존**
   - Phase 1에서 DendroNode → ACCNode 변환 함수 제공
   - Matrix 직접 입력 경로도 병존 (현행 acc_core_new.py 방식)

3. **acc_core_acc2.py는 독립 유지**
   - ACC2의 diameter 공식이 다르므로 ACCNode와 분리
   - 단, ACCNode 트리를 입력으로 받아 동심원 렌더링하는 방식으로 리팩토링 가능
