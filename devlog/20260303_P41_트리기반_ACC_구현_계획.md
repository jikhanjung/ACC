**날짜**: 2026-03-03

**관련**: R01 (ACC 자료구조 및 프로세스 설계), R02 (현행 구현 비교 분석)

## 목표

R01 설계 문서의 "트리 우선" ACC 알고리즘을 구현한다. 현행 `acc_core_new.py`의 greedy 알고리즘을 기반으로 하되, ACCNode 트리를 명시적으로 구축(sim, diameter, angle 저장)한 뒤 별도 단계로 좌표를 계산한다.

추가 요구사항:
- **Diversity index**: 각 area의 present taxa 수로 클러스터 내 area 순서(left/right) 결정
- **ACC2 탭 제거**: ACC2 기능 삭제, min/max diameter 설정만 ACC로 이전
- diameter 변경 시 트리 재구축 없이 좌표만 재계산

## Diversity Index 규칙

1. **Leaf 전용**: diversity는 area(leaf node)만 보유. Raw Data의 presence/absence 행렬에서 `sum(row)` = present taxa 수
2. **Raw Data 없으면**: 모든 area의 diversity를 동일하게(0) 설정 → ABC 순 정렬로 fallback
3. **두 area 합칠 때**: diversity가 높은 쪽이 LEFT
   - 동률이면 ABC 순 (먼저 오는 쪽이 LEFT)
4. **기존 클러스터에 area 추가**: 클러스터의 좌측 끝(leftmost leaf) vs 우측 끝(rightmost leaf)과 새 area의 similarity 비교
   - sim(new, left_edge) ≥ sim(new, right_edge) → 새 area가 LEFT
   - sim(new, right_edge) > sim(new, left_edge) → 새 area가 RIGHT
5. **두 클러스터 합칠 때**: 마주보는 끝 area의 similarity로 방향 결정

## 계획 항목

### Phase 1: `acc_core_tree.py` 신규 모듈

1. **ACCNode 데이터클래스**: members, left/right, local_sim, global_sim, diameter, angle, merge_order, diversity, points
   - `is_leaf`, `leftmost_leaf`, `rightmost_leaf`, `radius` 프로퍼티
2. **유틸 함수**: acc_core_new.py에서 복사 (pol2cart, cart2pol, get_similarity 등)
3. **`_determine_order()`**: diversity 규칙 + edge similarity 기반 left/right 결정
4. **`build_acc_tree()`**: greedy agglomeration으로 ACCNode 트리 구축
5. **`render_tree()`**: 좌표 계산 (min/max diameter 스케일링 포함)
6. **`generate_steps()`**: GUI용 progressive step snapshot 생성
7. **`build_acc_from_tree()`**: 최상위 API (build + render + steps)

### Phase 2: `acc_utils.py` 래퍼

- `build_acc_from_matrices_tree()`: 트리 기반 알고리즘 래퍼
- `rerender_acc_tree()`: 기존 트리에서 좌표만 재계산하는 래퍼

### Phase 3: GUI (`acc_gui.py`)

**ACC2 제거:**
- `from acc_core_acc2 import ...` 임포트 제거
- RightPanel의 QTabWidget, ACC2 탭 전체 삭제
- ACC2 관련 메서드 제거: `on_tab_changed`, `on_acc2_options_changed`, `generate_acc2`, `generate_acc2_with_options`
- ACCVisualizationWidget의 ACC2 메서드 11개 제거

**Diameter 컨트롤 추가:**
- Min/Max Diameter 입력 필드 (placeholder "auto")
- `on_diameter_changed()` → `MainWindow.rerender_acc()` 호출

**generate_acc() 교체:**
- Raw Data에서 diversity 계산
- `build_acc_from_matrices_tree()` 호출
- 트리 root와 merge_log 캐싱

**rerender_acc() 추가:**
- 캐시된 트리로 좌표만 재계산, step 위치 복원
