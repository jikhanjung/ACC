### 3. θ_btw 복원 버그 수정 (핵심)

**증상**: Simpson Arenig 데이터에서 최종 단계의 raw angle total이 462.63°로 계산됨 (정확한 값: 555.78°)

**원인 분석**:
- `_add_area`와 `_merge_clusters`에서 `theta_btw`를 원래 값으로 복원할 때, **closest pair**의 similarity를 사용
- 그러나 closest pair는 sequence 결정에만 사용되어야 하고, 실제 각도 복원에는 **경계 인접 쌍(boundary adjacent pair)**의 similarity를 사용해야 함
- 예: O를 추가할 때 closest pair = (J, O) sim=0.3077 → 124.62°, 하지만 실제 인접 쌍 = (O, Y) sim=0.1538 → 152.31°

**수정**:
```python
# 기존 (잘못된 코드)
theta_original = THETA_MAX_DEGREES * (1.0 - best_pair_sim)

# 수정 (올바른 코드)
btw_idx = _find_btw_index(...)
boundary_a = chosen_seq[btw_idx]
boundary_b = chosen_seq[btw_idx + 1]
boundary_sim = get_similarity(local_matrix, boundary_a, boundary_b)
theta_original = THETA_MAX_DEGREES * (1.0 - boundary_sim)
```

**검증**: raw total 555.78°, scale = 157.50 / 555.78 = 0.2834 (정확)

**변경 파일**:
- `acc_render_paper.py`: `_add_area`, `_merge_clusters`의 θ_btw 복원 로직
