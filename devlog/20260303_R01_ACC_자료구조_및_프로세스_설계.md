## 1. 목표

현행 ACC 구현은 dendrogram에서 클러스터를 추출한 뒤 **flat list**로 처리하면서 좌표 계산까지 동시에 수행한다. 이 접근에는 다음 문제가 있다:

- **구조와 렌더링이 혼재**: 클러스터 병합과 좌표 배치가 동일 루프에서 수행됨
- **트리 정보 소실**: 어떤 클러스터가 어떤 자식에서 파생되었는지 추적 불가
- **재렌더링 비용**: 파라미터 변경 시 전체 파이프라인 재실행 필요

**이상적 접근**:

1. 먼저 **트리 자료구조를 완성** — root(최종 병합)에서 leaf(개별 area)까지, 각 internal node에 `local_sim`, `global_sim`, `diameter`, `angle`을 기록
2. 그 다음 **트리로부터 ACC 다이어그램을 렌더링** — 자료구조가 완성되면 그리기는 단순 재귀 순회

이렇게 분리하면:

- 트리 구조를 독립적으로 검증/직렬화 가능
- 렌더링 파라미터(unit, 좌표계 등)를 변경해도 트리 재구축 불필요
- 디버깅 시 각 단계를 독립적으로 확인 가능

## 2. ACCNode 자료구조 정의

### 2.1 노드 타입

```
ACCNode
├── Leaf Node    — 개별 area (예: "J", "T", "Y")
├── Internal Node — 두 자식의 병합 결과
└── Root Node    — 최상위 Internal Node (모든 area 포함)
```

### 2.2 클래스 설계

```python
class ACCNode:
    """ACC 트리의 노드. Leaf이면 left/right가 None."""

    def __init__(self):
        # --- 구조 정보 (Phase 1에서 설정) ---
        self.members: set[str] = set()     # 이 노드가 포함하는 area 집합
        self.left: ACCNode | None = None   # 왼쪽 자식
        self.right: ACCNode | None = None  # 오른쪽 자식

        # --- 유사도 정보 (Phase 1~2에서 설정) ---
        self.local_sim: float = 1.0        # Local dendrogram 유사도
        self.global_sim: float = 1.0       # Global dendrogram/matrix 유사도

        # --- 기하학적 파라미터 (Phase 2에서 계산) ---
        self.diameter: float = 0.0         # unit / global_sim
        self.angle: float = 0.0            # 180° × (1 - local_sim)

        # --- 좌표 정보 (Phase 3에서 계산) ---
        self.center: tuple[float, float] = (0.0, 0.0)
        self.points: dict[str, tuple[float, float]] = {}  # member → (x, y)

    @property
    def is_leaf(self) -> bool:
        return self.left is None and self.right is None

    @property
    def is_root(self) -> bool:
        """외부에서 판단 (부모 참조가 없으므로 build 시 별도 관리)"""
        return len(self.members) > 1 and self.left is not None
```

### 2.3 Leaf Node vs Internal Node

| 속성 | Leaf Node | Internal Node |
|------|-----------|---------------|
| `members` | `{"J"}` (단일 원소) | `{"J", "T"}` 이상 |
| `left`, `right` | `None` | 자식 `ACCNode` |
| `local_sim` | `1.0` (자기 자신) | 해당 병합 단계의 유사도 |
| `global_sim` | `1.0` (자기 자신) | Global dendro/matrix에서 조회 |
| `diameter` | `0` (사용 안 함) | `unit / global_sim` |
| `angle` | `0` (사용 안 함) | `180° × (1 - local_sim)` |
| `points` | 자기 좌표 1개 | 모든 하위 member 좌표 |

### 2.4 트리 구조 예시

6개 area `{J, T, Y, N, O, Q}`에 대한 Local dendrogram이 아래와 같다고 가정:

```
Local dendrogram: (((J,T),Y),(N,(O,Q)))

             root [JTYNOQ]
            sim_local=0.6
           /              \
     [JTY]                 [NOQ]
   sim_local=0.8         sim_local=0.75
    /      \              /       \
  [JT]      Y           N        [OQ]
sim=0.9   (leaf)       (leaf)   sim=0.85
 / \
J   T
```

완성된 ACCNode 트리:

```
ACCNode(root)
├── members = {J, T, Y, N, O, Q}
├── local_sim = 0.6
├── global_sim = 0.55  ← Global dendro에서 조회
├── diameter = 1.0 / 0.55 = 1.818
├── angle = 180° × (1 - 0.6) = 72°
├── left → ACCNode
│   ├── members = {J, T, Y}
│   ├── local_sim = 0.8
│   ├── global_sim = 0.78
│   ├── diameter = 1.282
│   ├── angle = 36°
│   ├── left → ACCNode
│   │   ├── members = {J, T}
│   │   ├── local_sim = 0.9
│   │   ├── global_sim = 0.88
│   │   ├── diameter = 1.136
│   │   ├── angle = 18°
│   │   ├── left → ACCNode(leaf, members={J})
│   │   └── right → ACCNode(leaf, members={T})
│   └── right → ACCNode(leaf, members={Y})
└── right → ACCNode
    ├── members = {N, O, Q}
    ├── local_sim = 0.75
    ├── global_sim = 0.70
    ├── diameter = 1.429
    ├── angle = 45°
    ├── left → ACCNode(leaf, members={N})
    └── right → ACCNode
        ├── members = {O, Q}
        ├── local_sim = 0.85
        ├── global_sim = 0.83
        ├── diameter = 1.205
        ├── angle = 27°
        ├── left → ACCNode(leaf, members={O})
        └── right → ACCNode(leaf, members={Q})
```

## 3. 구축 프로세스 (3단계)

### Phase 1: 트리 골격 추출

**입력**: Local dendrogram (`DendroNode` 트리)
**출력**: `ACCNode` 트리 (구조 + `local_sim`만 설정)

```python
def build_tree_from_dendro(dendro_root: DendroNode) -> ACCNode:
    """Local dendrogram을 ACCNode 트리로 변환"""
    node = ACCNode()
    node.members = set(dendro_root.members)
    node.local_sim = dendro_root.sim

    if dendro_root.left is not None:
        node.left = build_tree_from_dendro(dendro_root.left)
    if dendro_root.right is not None:
        node.right = build_tree_from_dendro(dendro_root.right)

    return node
```

이 단계 후 트리 상태:

```
각 노드: members ✓, local_sim ✓, left/right ✓
          global_sim ✗, diameter ✗, angle ✗, points ✗
```

### Phase 2: 속성 계산 (Decoration)

**입력**: Phase 1의 ACCNode 트리 + Global dendrogram + Global similarity matrix + unit
**출력**: 각 internal node에 `global_sim`, `diameter`, `angle` 설정

```python
def decorate_tree(node: ACCNode, global_dendro, global_matrix, unit=1.0):
    """트리를 후위 순회하며 global_sim, diameter, angle 계산"""
    if node.is_leaf:
        return  # Leaf는 계산 불필요

    # 자식 먼저 처리 (후위 순회)
    decorate_tree(node.left, global_dendro, global_matrix, unit)
    decorate_tree(node.right, global_dendro, global_matrix, unit)

    # Global similarity 조회
    global_sim = find_in_global_dendro(global_dendro, node.members)
    if global_sim is None:
        global_sim = average_pairwise_similarity(node.members, global_matrix)

    node.global_sim = global_sim

    # 기하학적 파라미터 계산
    node.diameter = unit / global_sim if global_sim > 0 else unit * 1000
    node.angle = 180.0 * (1.0 - node.local_sim)
```

이 단계 후 트리 상태:

```
각 internal 노드: members ✓, local_sim ✓, global_sim ✓,
                  diameter ✓, angle ✓
                  points ✗ (아직 좌표 미계산)
```

### Phase 3: 좌표 계산 및 렌더링

**입력**: Phase 2의 완성된 ACCNode 트리
**출력**: 각 leaf node에 (x, y) 좌표 할당

좌표 계산은 **루트에서 리프 방향** (전위 순회)으로 수행한다:

```python
def compute_coordinates(node: ACCNode, parent_direction=0.0):
    """루트→리프 방향으로 좌표 계산 (전위 순회)"""
    if node.is_leaf:
        # Leaf: 부모 방향에 따라 반지름 위에 배치
        r = ???  # 부모의 radius
        node.points[next(iter(node.members))] = pol2cart(r, parent_direction)
        return

    radius = node.diameter / 2.0
    half_angle = node.angle / 2.0

    # 왼쪽 자식: parent_direction - half_angle
    left_dir = parent_direction - half_angle
    # 오른쪽 자식: parent_direction + half_angle
    right_dir = parent_direction + half_angle

    compute_coordinates(node.left, left_dir)
    compute_coordinates(node.right, right_dir)

    # 이 노드의 points = 왼쪽 points + 오른쪽 points
    node.points = {**node.left.points, **node.right.points}
```

## 4. 기하학적 공식

### 4.1 핵심 공식

| 파라미터 | 공식 | 의미 |
|----------|------|------|
| **Diameter (d)** | `d = unit / sim_global` | Global 유사도의 역수. 유사도 높을수록 원 작아짐 |
| **Angle (θ)** | `θ = 180° × (1 - sim_local)` | Local 유사도의 보수. 유사도 높을수록 각도 좁아짐 |
| **Radius (r)** | `r = d / 2` | 원의 반지름 |

### 4.2 좌표 변환

극좌표 → 직교좌표:
```
x = r × cos(θ)
y = r × sin(θ)
```

### 4.3 유사도와 기하 파라미터 관계

```
sim_global  →  diameter  →  원의 크기
   1.0          1.0         가장 작음 (밀접)
   0.5          2.0         중간
   0.1         10.0         매우 큼 (느슨)

sim_local   →  angle     →  부채꼴 각도
   1.0           0°        겹침 (동일)
   0.5          90°        직각
   0.0         180°        반대편
```

### 4.4 ACC2 변형 공식

ACC2에서는 다른 diameter 공식을 사용한다:

```
diameter = 1 + (1 - sim_global)    # ACC2 전용
```

이 공식은 `sim_global=1.0`일 때 `d=1.0`, `sim_global=0.0`일 때 `d=2.0`으로 범위가 제한된다.

## 5. 렌더링 알고리즘

### 5.1 재귀적 좌표 배치 (루트 → 리프)

렌더링은 완성된 트리를 **전위 순회**하며 수행한다:

```
render(root):
    1. root의 center = (0, 0)
    2. root의 radius = root.diameter / 2
    3. root의 angle을 좌/우 자식에 분배:
       - left child 방향 = parent_direction - angle/2
       - right child 방향 = parent_direction + angle/2
    4. 각 자식에 대해 재귀:
       - leaf이면: 해당 방향 + 부모 radius 위에 점 찍기
       - internal이면: 해당 방향을 중심으로 자신의 angle 분배
```

### 5.2 원(circle) 그리기

각 internal node는 하나의 원을 그린다:
- 중심: (0, 0) — 모든 원은 원점 중심 (동심원)
- 반지름: `node.diameter / 2`

```
root      → 가장 큰 원 (전체)
  ├── JTY → 중간 원
  │   └── JT → 작은 원
  └── NOQ → 중간 원
      └── OQ → 작은 원
```

### 5.3 ACC1 vs ACC2 렌더링 차이

| 항목 | ACC1 | ACC2 |
|------|------|------|
| Area 위치 | 각 클러스터의 고유 반지름 위 | 모두 r=0.5 (최내곽 원) |
| 원의 의미 | 클러스터의 크기 표현 | 병합 단계별 동심원 |
| 연결선 | 없음 | Radial 선 + Arc (dendrogram 투영) |
| diameter 공식 | `unit / sim_global` | `1 + (1 - sim_global)` |

## 6. ASCII 다이어그램

### 6.1 ACCNode 트리 → ACC 다이어그램 변환 과정

```
[입력: 두 개의 Dendrogram + Matrix]

         Local Dendro              Global Dendro
        (구조 + sim_L)            (sim_G 조회용)
              │                        │
              ▼                        │
   ┌──────────────────┐                │
   │  Phase 1:        │                │
   │  트리 골격 추출   │                │
   │  (DendroNode →   │                │
   │   ACCNode)       │                │
   └────────┬─────────┘                │
            │ ACCNode 트리              │
            │ (sim_local만 설정)        │
            ▼                          ▼
   ┌──────────────────────────────────────┐
   │  Phase 2: 속성 계산 (Decoration)     │
   │  - global_sim = 조회(global dendro)  │
   │    없으면 → average(global matrix)   │
   │  - diameter = unit / global_sim      │
   │  - angle = 180° × (1 - local_sim)    │
   └────────────────┬─────────────────────┘
                    │ ACCNode 트리
                    │ (sim, d, θ 모두 설정)
                    ▼
   ┌──────────────────────────────────────┐
   │  Phase 3: 좌표 계산 + 렌더링         │
   │  - 루트→리프 전위 순회               │
   │  - 각 노드: 방향 = 부모방향 ± θ/2   │
   │  - leaf: (x,y) = pol2cart(r, 방향)   │
   └────────────────┬─────────────────────┘
                    │
                    ▼
           [ACC 다이어그램 출력]
```

## 7. 정리

### 이 설계의 핵심 원칙

1. **자료구조 우선**: 먼저 완전한 트리를 구축하고, 렌더링은 별도 단계로 분리
2. **관심사 분리**: 구조(Phase 1) → 속성(Phase 2) → 좌표(Phase 3)
3. **불변성**: Phase 1~2의 트리는 렌더링 파라미터와 독립적
4. **검증 용이**: 각 Phase 완료 후 트리 상태를 독립적으로 검증 가능

### 현행 구현과의 관계

이 설계가 현행 구현(acc_core.py, acc_core_new.py, acc_core_acc2.py)과 어떻게 다른지는 **R02 문서**에서 상세히 비교한다.
